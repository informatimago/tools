* How this tracker works

Each issue is one file in =issues/open/= (or =issues/closed/= once resolved)
with the =.issue= extension and an org-mode local-variables line.  When an issue
is resolved, move it to =issues/closed/= (=git mv=) with a RESOLUTION note and
tick its TODO -> DONE.

Issues were produced by a full-tree code review on 2026-06-20 (gcc/g++ 14,
the makedir strict flag set), building every module against the freshly
reinstalled =common= libraries under =/usr/local/lib=.  Severities: =critical=
(memory-unsafe or silently wrong), =high=, =medium=, =low=.  All findings below
are *proposed corrections awaiting review/validation* — nothing has been changed
in the sources yet.
